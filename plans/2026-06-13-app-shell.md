## Context

`feat/web-foundation` delivered the design system and a locked, in-game-faithful skills panel, but there is no page yet — the skills panel only exists inside the component showcase (`web/components.html`). This plan builds the **app shell**: the real page scaffold that every later brick (collection log, goal tracker, …) slots into. It is the first time the project reads as a product rather than a component gallery.

## Goals

- A real entry page `web/index.html` with a header, a two-column body, and a footer.
- **Header:** the "Gilded Ledger" logo/wordmark, the Tiger0295 identity (name, ironman indicator, combat/total/XP), and a small set of resource/stat counters.
- **Body layout:** an authentic **left "mirror" column** and a modern **right "tracker" column**, responsive — they stack vertically on mobile (≤640px).
- The **locked skills panel** rendered in the left mirror, fed by the same baked Tiger0295 snapshot the showcase uses (live data is a later brick).
- **Footer:** the Jagex IP fan-project disclaimer.
- Reuse existing tokens/components only; introduce no new dependencies; mobile-first and touch-friendly.

## Non-goals

- No live data / backend (that is `feat/backend-ingest` + `feat/api`).
- No collection-log, quests, diaries, CA, or goal-tracker content (later bricks) — the tracker column shows a labeled placeholder.
- No real player-model avatar render (see [ADR-0001](../decisions/2026-06-13-0001-web-app-first.md) note) — use a placeholder ironman-helm avatar in a ring; upgrade when the plugin lands.
- No routing/multi-page.

## Key decisions

- **Layout:** CSS grid, `grid-template-columns: minmax(0, <mirror>) minmax(0, 1fr)` on desktop, single column ≤640px. Mirror column is a fixed-ish width sized to the authentic panels; tracker column is fluid.
- **Header** is its own component (`web/styles/` addition, e.g. `.appbar`), built from existing tokens; avatar uses the `slot`/ring treatment from the component library.
- **Avatar:** static OSRS ironman-helm sprite for now (self-hosted), in a colored ring — placeholder, not the RuneProfile 3D render.
- Skills panel markup/JS is **extracted from `components.html` into a reusable include** (a small JS render module under `web/`) so `index.html` and the showcase share one source of truth — no copy-paste divergence.

## Intended changes

- **Add** `web/index.html` — the app shell (header + grid body + footer).
- **Add** `web/styles/app.css` — appbar/header, layout grid, footer styles (tokens only).
- **Add** `web/scripts/skills-panel.js` — the skills-panel data + render extracted from `components.html`, importable by both pages.
- **Modify** `web/components.html` — consume `skills-panel.js` instead of its inline copy (no visual change).
- **Add** a self-hosted ironman-helm avatar sprite under `web/assets/sprites/`.
- **Modify** [SPEC.md](../SPEC.md) roadmap row for `feat/app-shell` → status `shipped` on merge.

## Verification

- `web/index.html` opens via the tunnel on desktop **and** mobile; no console errors; no horizontal overflow at 320px, 390px, 768px, 1280px.
- Header shows Tiger0295 identity + counters; footer shows the Jagex disclaimer.
- Two-column layout on desktop; **stacks** to one column ≤640px (mirror above tracker).
- The skills panel renders identically in `index.html` and `components.html` (shared module — confirm by diffing the rendered panel).
- Tracker column shows a labeled "Goal tracker — coming next" placeholder.
- `node --check web/scripts/skills-panel.js` passes; the panel still shows the XP tooltip (hover + tap) and virtual levels.
